1. Ringkasan Aplikasi

NoteUang Me adalah aplikasi pencatatan keuangan pribadi untuk mencatat pemasukan dan pengeluaran, memantau saldo bersih bulanan, membuat anggaran per kategori, mengelola target keuangan, menampilkan insight keuangan, serta menyiapkan laporan transaksi dalam format Excel.

Data utama aplikasi disimpan lokal di perangkat menggunakan SQLite. Preferensi seperti sesi login, tema gelap, konfigurasi AI, API key AI, dan riwayat notifikasi disimpan menggunakan SharedPreferences. Fitur AI bersifat opsional; jika API key tidak tersedia atau panggilan AI gagal, aplikasi tetap memberikan analisis lokal.

2. Tujuan Bisnis

  • Membantu pengguna mencatat transaksi keuangan harian secara terstruktur.
  • Menghitung total pemasukan, total pengeluaran, dan saldo bersih dalam periode aktif.
  • Mengontrol pengeluaran melalui anggaran bulanan per kategori.
  • Membantu pengguna membuat dan memantau goal keuangan seperti dana darurat, liburan, rumah, atau investasi.
  • Menyediakan saran keuangan berbasis data transaksi, anggaran, dan goal.
  • Membuat laporan transaksi yang dapat dibagikan dalam format Excel.
  • Memberikan notifikasi saat data berhasil disimpan dan saat saldo bulan berjalan berada pada kondisi negatif.

3. Aktor dan Peran

Aktor Peran Interaksi Utama
Pengguna lokal Pemilik data keuangan pribadi. Register, login, mencatat transaksi, mengatur kategori, membuat anggaran, membuat goal, melihat dashboard, ekspor laporan, mengatur AI dan tema.
Sistem aplikasi Memproses data, validasi, kalkulasi, penyimpanan, dan refresh tampilan. Menghitung saldo, agregasi kategori, budget usage, progress goal, prediksi lokal, dan validasi form.
Layanan perangkat Menyediakan kemampuan native perangkat. SQLite, SharedPreferences, notifikasi lokal, kamera, galeri, OCR ML Kit, share file, dan WhatsApp link.
OpenAI API Layanan eksternal opsional untuk analisis AI. Dipakai hanya jika pengguna menyimpan API key dan mengaktifkan Analisis AI.

4. Modul dan Data Utama

Modul Data Utama Penyimpanan Keterangan Proses
Autentikasi Nama lengkap, email, nomor WhatsApp, password hash, tanggal dibuat, sesi user. Tabel users dan SharedPreferences auth_session_user_id. Register membuat user lokal pertama atau tambahan. Login memeriksa email dan password hash. Logout menghapus sesi.
Kategori Nama kategori, tipe pemasukan/pengeluaran, ikon, warna. Tabel categories. Kategori awal otomatis dibuat saat database pertama kali dibuat. Kategori yang sudah digunakan transaksi atau anggaran tidak boleh dihapus.
Transaksi Nominal, tipe, kategori, tanggal, catatan. Tabel transactions. Transaksi dapat dibuat, diedit, dihapus, difilter, dilihat detailnya, discan dari struk, dan diekspor ke Excel.
Anggaran Kategori pengeluaran, bulan, tahun, batas anggaran. Tabel budgets. Satu kategori hanya punya satu anggaran untuk bulan dan tahun yang sama. Jika data sama disimpan lagi, sistem melakukan update.
Goal Keuangan Nama goal, target nominal, tanggal target, jumlah terkumpul. Tabel goals. Sistem menghitung progress, sisa target, bulan tersisa, dan estimasi setoran bulanan.
Dashboard Total pemasukan, total pengeluaran, saldo bersih, pengeluaran per kategori, perbandingan bulanan, budget usage, goal. Data agregat dari transaksi, budget, goal, dan kategori. Dashboard memuat data bulan berjalan dan grafik ringkasan untuk pengambilan keputusan cepat.
Advisor Saran keuangan, projected balance, engine analisis. Data dihitung dari transaksi, budget, goal, kategori, dan preferensi AI. Analisis lokal selalu tersedia. Analisis AI dipakai jika API key dan toggle AI aktif.
Notifikasi Judul, isi, waktu, payload, status saldo negatif harian. SharedPreferences dan local notification plugin. Notifikasi disimpan sebagai riwayat maksimal 50 item dan ditampilkan di dashboard.

5. Proses Bisnis Utama

5.1 Proses Awal Aplikasi

  1. Aplikasi menjalankan inisialisasi Flutter, locale Indonesia id_ID, dependency injection, database SQLite, dan service notifikasi.
  2. Sistem memuat cubit utama: Auth, Settings, Category, Transaction, Budget, Goal, Dashboard, dan Advisor.
  3. Splash screen tampil minimal 1,2 detik saat status autentikasi diperiksa.
  4. Jika belum ada user lokal, aplikasi membuka halaman Register.
  5. Jika user sudah ada dan session user valid, aplikasi membuka Main Navigation.
  6. Jika user sudah ada tetapi session kosong, aplikasi membuka halaman Login.

5.2 Proses Register dan Login

  1. Pengguna mengisi nama lengkap, nomor WhatsApp, email, dan kata sandi saat register.
  2. Sistem memvalidasi nama minimal 3 karakter, nomor WhatsApp minimal 10 digit, email berformat dasar valid, dan kata sandi minimal 6 karakter.
  3. Sistem menormalisasi email menjadi huruf kecil untuk pengecekan duplikasi.
  4. Sistem menormalisasi nomor WhatsApp: awalan 0 diubah menjadi awalan 62.
  5. Password disimpan sebagai hash SHA-256 di database lokal.
  6. Setelah register berhasil, session user disimpan di SharedPreferences dan pengguna langsung masuk ke aplikasi.
  7. Pada login, sistem mencari email lokal dan membandingkan hash password.
  8. Logout menghapus session user dan mengembalikan aplikasi ke halaman Login.

5.3 Proses Kelola Kategori

  1. Saat database dibuat pertama kali, aplikasi membuat kategori awal untuk pengeluaran dan pemasukan.
  2. Pengguna dapat membuka Pengaturan lalu Kelola Kategori.
  3. Pengguna dapat menambah kategori dengan nama, tipe, ikon, dan warna.
  4. Pengguna dapat mengedit kategori yang ada.
  5. Saat hapus kategori, sistem memeriksa apakah kategori sudah dipakai transaksi atau anggaran.
  6. Jika kategori sedang dipakai, penghapusan diblokir untuk menjaga konsistensi data.

5.4 Proses Pencatatan Transaksi Manual

  1. Pengguna menekan tombol tambah di bottom navigation atau tombol Tambah pada halaman Transaksi.
  2. Pengguna memilih tipe transaksi: Pemasukan atau Pengeluaran.
  3. Sistem menampilkan kategori yang sesuai dengan tipe transaksi.
  4. Pengguna mengisi nominal, memilih kategori, memilih tanggal, dan menambahkan catatan opsional.
  5. Sistem memvalidasi nominal lebih dari 0 dan kategori wajib dipilih.
  6. Transaksi disimpan ke SQLite.
  7. Sistem memuat ulang transaksi, dashboard, advisor, dan anggaran bulan aktif.
  8. Sistem menampilkan notifikasi lokal bahwa transaksi berhasil disimpan.

5.5 Proses Scan Struk OCR

  1. Pengguna menekan Scan Struk dari halaman Transaksi atau ikon scanner pada form transaksi.
  2. Sistem memastikan fitur OCR hanya berjalan pada Android dan iOS.
  3. Pengguna memilih sumber gambar: kamera atau galeri.
  4. Aplikasi membaca teks pada gambar menggunakan Google ML Kit Text Recognition.
  5. Sistem mencoba mendeteksi nominal terbaik dari teks dengan prioritas kata kunci seperti total, jumlah, bayar, tagihan, amount, subtotal, Rp, dan IDR.
  6. Jika nominal berhasil terdeteksi, nilai otomatis dimasukkan ke form.
  7. Jika gagal, pengguna dapat mengisi nominal manual dari bottom sheet konfirmasi.
  8. Pengguna melanjutkan penyimpanan transaksi seperti proses manual.

5.6 Proses Filter, Detail, Edit, Hapus, dan Ekspor Transaksi

  1. Halaman Transaksi menampilkan saldo periode aktif, total pemasukan, total pengeluaran, dan jumlah data.
  2. Pengguna dapat memilih periode: bulan ini, bulan lalu, custom range, atau semua transaksi.
  3. Pengguna dapat memfilter transaksi berdasarkan kategori.
  4. Transaksi dikelompokkan berdasarkan tanggal dan ditampilkan dari tanggal terbaru.
  5. Tap transaksi membuka detail transaksi.
  6. Dari detail, pengguna dapat mengedit atau menghapus transaksi.
  7. Sebelum menghapus, aplikasi meminta konfirmasi.
  8. Pengguna dapat membagikan laporan Excel untuk transaksi yang sedang aktif berdasarkan filter.
  9. Jika tidak ada transaksi pada filter aktif, aplikasi menolak ekspor dan menampilkan pesan.

5.7 Proses Dashboard dan Insight

  1. Dashboard mengambil data bulan berjalan.
  2. Sistem menghitung total pemasukan, total pengeluaran, dan saldo bersih.
  3. Sistem menghitung pengeluaran per kategori untuk pie chart.
  4. Sistem mengambil perbandingan pemasukan dan pengeluaran beberapa bulan terakhir untuk bar chart.
  5. Sistem menghitung pemakaian anggaran per kategori.
  6. Sistem menampilkan 5 transaksi terbaru.
  7. Sistem menampilkan banner insight dari Advisor.
  8. Jika saldo bulan berjalan negatif, sistem mengirim notifikasi kondisi keuangan tidak sehat maksimal satu kali per hari.
  9. Jika pengguna memiliki nomor WhatsApp, sistem membuka link WhatsApp berisi ringkasan kondisi keuangan negatif.

5.8 Proses Anggaran Bulanan

  1. Pengguna membuka tab Anggaran.
  2. Pengguna memilih bulan dengan tombol bulan sebelumnya atau bulan berikutnya.
  3. Pengguna menekan Tambah Anggaran.
  4. Pengguna memilih kategori pengeluaran dan mengisi batas anggaran.
  5. Sistem memvalidasi kategori wajib dan nominal anggaran lebih dari 0.
  6. Sistem menyimpan anggaran atau memperbarui anggaran yang sudah ada untuk kategori, bulan, dan tahun yang sama.
  7. Sistem menghitung total pengeluaran kategori tersebut pada bulan aktif.
  8. Sistem menampilkan progress pemakaian, sisa anggaran, dan status melalui komponen budget bar.
  9. Pengguna dapat mengedit atau menghapus anggaran setelah konfirmasi.

5.9 Proses Goal Keuangan

  1. Pengguna membuka tab Goals.
  2. Pengguna menekan Tambah Goal.
  3. Pengguna mengisi nama goal, target nominal, jumlah terkumpul, dan tanggal target.
  4. Sistem memvalidasi nama wajib, target lebih dari 0, jumlah terkumpul tidak negatif, dan tanggal target tidak kurang dari hari ini.
  5. Sistem menghitung persentase progress, nominal tersisa, bulan tersisa, dan saran setoran bulanan.
  6. Pengguna dapat menambah alokasi tabungan ke goal yang sudah ada.
  7. Pengguna dapat mengedit atau menghapus goal setelah konfirmasi.
  8. Setelah goal berubah, dashboard dan advisor dimuat ulang.

5.10 Proses Analisis Lokal dan AI

  1. Advisor mengumpulkan transaksi, budget usage, goal, kategori pengeluaran, pengeluaran bulan berjalan, dan perbandingan 3 bulan terakhir.
  2. Analisis lokal membuat saran berdasarkan dominasi kategori, anggaran yang mendekati habis atau terlampaui, cashflow rata-rata, dan kemampuan mengejar goal.
  3. Jika tidak ada data cukup, sistem menyarankan pengguna membangun riwayat transaksi.
  4. Jika API key AI tersedia dan toggle AI aktif, sistem mengirim payload ringkasan ke OpenAI Chat Completions API.
  5. Respons AI wajib berupa JSON berbahasa Indonesia dengan projected balance, summary, dan daftar advice.
  6. Jika AI gagal, aplikasi otomatis memakai analisis lokal dan tidak memblokir pengalaman pengguna.

5.11 Proses Pengaturan

  1. Pengguna membuka tab Pengaturan.
  2. Pengguna dapat mengaktifkan atau menonaktifkan dark mode.
  3. Pengguna dapat membuka konfigurasi AI, mengisi API key, memilih model, mengaktifkan AI, atau menghapus API key lokal.
  4. Pengguna dapat membuka halaman Kelola Kategori.
  5. Pengguna dapat melihat identitas user lokal.
  6. Pengguna dapat logout dari sesi aktif.

8. Panduan Penggunaan Lengkap

8.1 Membuka Aplikasi Pertama Kali

  1. Buka aplikasi NoteUang Me.
  2. Tunggu halaman Splash selesai memuat.
  3. Jika belum ada akun, aplikasi langsung menampilkan form register.
  4. Isi nama lengkap, nomor WhatsApp, email, dan kata sandi.
  5. Tekan Daftar Sekarang.
  6. Setelah berhasil, pengguna masuk ke Dashboard.

8.2 Login

  1. Buka aplikasi.
  2. Jika sesi tidak aktif, halaman Login ditampilkan.
  3. Masukkan email dan kata sandi yang sudah terdaftar di perangkat.
  4. Tekan Login.
  5. Jika email atau kata sandi salah, aplikasi menampilkan pesan kesalahan.

8.3 Melihat Dashboard

  1. Buka tab Dashboard.
  2. Lihat saldo bersih, total pemasukan, dan total pengeluaran bulan berjalan.
  3. Lihat shortcut kategori pengeluaran untuk membuka transaksi kategori tersebut.
  4. Baca insight atau saran keuangan dari kartu Advisor.
  5. Lihat transaksi terbaru.
  6. Lihat progress anggaran, grafik kategori pengeluaran, dan grafik pemasukan versus pengeluaran.
  7. Tarik layar ke bawah untuk refresh data.
  8. Tekan ikon notifikasi untuk melihat riwayat notifikasi.

8.4 Menambah Transaksi Manual

  1. Tekan tombol plus di bottom navigation atau tekan Tambah pada halaman Transaksi.
  2. Pilih tipe transaksi: Pengeluaran atau Pemasukan.
  3. Masukkan nominal transaksi.
  4. Pilih kategori yang sesuai.
  5. Pilih tanggal transaksi.
  6. Isi catatan jika diperlukan.
  7. Tekan Simpan Transaksi.
  8. Aplikasi menyimpan data, memperbarui dashboard, dan menampilkan notifikasi transaksi tersimpan.

8.5 Menambah Transaksi dengan Scan Struk

  1. Buka halaman Transaksi.
  2. Tekan Scan Struk.
  3. Pilih Ambil Foto untuk membuka kamera atau Upload dari Galeri untuk memilih foto struk.
  4. Berikan izin kamera atau galeri jika diminta perangkat.
  5. Tunggu proses OCR membaca teks pada gambar.
  6. Jika nominal terbaca, nominal otomatis masuk ke form transaksi.
  7. Jika nominal belum terbaca, isi nominal secara manual pada form konfirmasi.
  8. Lengkapi kategori, tanggal, dan catatan, lalu simpan transaksi.

8.6 Melihat, Memfilter, dan Mengelola Transaksi

  1. Buka tab Transaksi.
  2. Pilih periode cepat: Bulan ini, Bulan lalu, Custom, atau Semua.
  3. Gunakan filter untuk memilih periode dan kategori secara lebih detail.
  4. Tekan salah satu transaksi untuk melihat detail.
  5. Pilih Edit Transaksi untuk mengubah data.
  6. Pilih Hapus untuk menghapus transaksi setelah konfirmasi.
  7. Tekan ikon share untuk membuat dan membagikan laporan Excel berdasarkan filter aktif.

8.7 Membuat Laporan Excel

  1. Buka halaman Transaksi.
  2. Atur filter transaksi sesuai kebutuhan laporan.
  3. Tekan ikon share.
  4. Aplikasi membuat file Excel berisi periode, waktu dibuat, total pemasukan, total pengeluaran, saldo bersih, dan daftar transaksi.
  5. Pilih aplikasi tujuan dari share sheet perangkat.
  6. Jika tidak ada transaksi pada filter aktif, aplikasi tidak membuat laporan dan menampilkan pesan.

8.8 Membuat dan Mengelola Anggaran

  1. Buka tab Anggaran.
  2. Pilih bulan yang ingin diatur menggunakan tombol kiri atau kanan.
  3. Tekan Tambah Anggaran.
  4. Pilih kategori pengeluaran.
  5. Isi batas anggaran dalam Rupiah.
  6. Tekan Simpan.
  7. Lihat progress pemakaian anggaran pada daftar.
  8. Gunakan tombol edit atau hapus pada item anggaran jika diperlukan.

8.9 Membuat dan Mengelola Goal Keuangan

  1. Buka tab Goals.
  2. Tekan Tambah Goal.
  3. Isi nama goal, target jumlah, jumlah terkumpul, dan tanggal target.
  4. Lihat preview progress dan estimasi setoran bulanan.
  5. Tekan Simpan Goal.
  6. Untuk menambah progress, tekan tambah alokasi pada goal lalu isi nominal.
  7. Gunakan edit untuk mengubah goal atau hapus untuk menghapus goal setelah konfirmasi.

8.10 Mengelola Kategori

  1. Buka tab Pengaturan.
  2. Pilih Kelola Kategori.
  3. Tekan tombol Kategori untuk menambah kategori baru.
  4. Isi nama kategori, tipe pemasukan atau pengeluaran, ikon, dan warna.
  5. Tekan Simpan.
  6. Gunakan menu pada item kategori untuk edit atau hapus.
  7. Jika kategori sudah dipakai transaksi atau anggaran, aplikasi menolak penghapusan.

8.11 Mengatur AI Advisor

  1. Buka tab Pengaturan.
  2. Pilih Konfigurasi AI.
  3. Masukkan API key.
  4. Isi model AI, default aplikasi adalah gpt-4o-mini.
  5. Aktifkan toggle Analisis AI.
  6. Tekan Simpan Konfigurasi AI.
  7. Kembali ke dashboard atau goal untuk melihat advisor diperbarui.
  8. Jika ingin berhenti memakai AI, hapus API key atau nonaktifkan toggle.

8.12 Mengatur Tema dan Logout

  1. Buka tab Pengaturan.
  2. Gunakan switch Dark mode untuk mengganti tema aplikasi.
  3. Lihat kartu user untuk memastikan akun lokal yang sedang aktif.
  4. Tekan Logout untuk keluar dari sesi saat ini.

11. Notifikasi dan Riwayat

  • Aplikasi meminta izin notifikasi pada inisialisasi service notifikasi.
  • Notifikasi input tersimpan muncul saat akun, transaksi, kategori, budget, atau goal berhasil disimpan.
  • Notifikasi saldo negatif muncul saat saldo bersih bulan berjalan lebih kecil dari 0.
  • Riwayat notifikasi disimpan di SharedPreferences dengan batas maksimal 50 item terbaru.
  • Riwayat dapat dilihat dari dashboard melalui bottom sheet Riwayat Notifikasi.

12. Laporan Excel

Laporan Excel dibuat dari halaman Transaksi menggunakan package excel dan dibagikan dengan share_plus. File disimpan sementara dengan nama seperti report_keuangan_yyyyMMdd_HHmmss.xlsx. Isi laporan meliputi judul, periode, waktu dibuat, total pemasukan, total pengeluaran, saldo bersih, serta daftar transaksi dengan kolom tanggal, jenis, kategori, nominal, dan catatan.

15. Catatan Keamanan dan Batasan

  • Data aplikasi tersimpan lokal di perangkat; tidak ada sinkronisasi cloud pada kode saat ini.
  • API key AI disimpan lokal di SharedPreferences. Untuk produksi, penyimpanan aman berbasis secure storage lebih disarankan.
  • Password user disimpan sebagai hash SHA-256 lokal. Untuk produksi, password sebaiknya memakai password hashing khusus seperti Argon2, bcrypt, atau PBKDF2 dengan salt.
  • Halaman receipt tersedia di kode, tetapi tidak terlihat dipanggil dari proses simpan transaksi saat ini.
  • OCR bergantung pada kualitas gambar struk, izin kamera/galeri, dan dukungan platform Android atau iOS.
  • Analisis AI membutuhkan koneksi internet dan API key valid. Jika tidak tersedia, aplikasi tetap memakai analisis lokal.
